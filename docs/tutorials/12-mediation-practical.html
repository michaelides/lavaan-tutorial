Mediation — practical

Path mediation then latent SEM mediation with bootstrap BCAs

Author
Affiliation

George Michaelides

School of Psychology, University of East Anglia

Published

January 1, 2026

Learning objectives

By the end of this tutorial you will be able to:

  • label regression paths in lavaan with * and define the indirect effect with :=
  • fit a path mediation model with bootstrap standard errors and extract BCa confidence intervals
  • extend the mediation model to a full latent-variable SEM with three factors
  • report mediation results in APA format, including the indirect effect and its confidence interval

Setup

Setup code (you have seen this before) 
library(haven)
library(lavaan)
library(tidySEM)

The mediation model

We test whether positive affect (PA) mediates the relationship between job control and creativity:

  • \(X\) = job control (con)
  • \(M\) = positive affect (pa)
  • \(Y\) = creativity (creat)

We fit this model in two forms:

  1. Path mediation — manifest variables (mean scores), as a direct extension of Tutorial 09
  2. Latent SEM mediation — latent variables with item-level indicators, as an extension of Tutorial 10

Both use the same := syntax to define the indirect effect.

Worked example 1: Path mediation with manifest variables

Computing the manifest variables

ps$con   <- (ps$con1 + ps$con3 + ps$con6 + ps$con8 + ps$con9 + ps$con10) / 6
ps$pa    <- (ps$pa1 + ps$pa2 + ps$pa3) / 3
ps$creat <- (ps$creat1 + ps$creat2 + ps$creat3) / 3

Specifying the model with labelled paths

The key innovation is labelling the regression paths with * so we can refer to them in := definitions:

model_med_path <- '
  creat ~ b * pa + c * con
  pa    ~ a * con

  indirect := a * b
  direct   := c
  total    := c + (a * b)
'

Reading the syntax:

  • creat ~ b * pa + c * con — the path from pa to creat is labelled b; the path from con to creat is labelled c
  • pa ~ a * con — the path from con to pa is labelled a
  • indirect := a * b — defines the indirect effect as the product \(ab\)
  • direct := c — defines the direct effect as \(c'\)
  • total := c + (a * b) — defines the total effect as \(c' + ab\)

Fitting with bootstrap standard errors

To test the indirect effect, we use bootstrap confidence intervals. This requires two changes:

  1. Switch the estimator from MLR to ML (MLR is incompatible with se = "boot")
  2. Request bootstrap standard errors with se = "boot" and bootstrap = 1000
set.seed(1234)
fit_med_path <- sem(model_med_path, data = ps,
                    estimator = "ML", se = "boot", bootstrap = 1000)
ImportantEstimator constraint with bootstrapping

When you request se = "boot", lavaan requires estimator = "ML" — the robust MLR estimator cannot be combined with bootstrapping. This is because the bootstrap builds its own empirical sampling distribution, making the Satorra–Bentler scaling correction unnecessary. The point estimates are the same under ML and MLR; only the standard errors differ, and for the indirect effect we use the bootstrap standard errors.

Extracting bootstrap confidence intervals

For clean BCa confidence intervals, use parameterEstimates() with boot.ci.type = "bca.simple":

pe_med <- parameterEstimates(fit_med_path, standardized = TRUE,
                             boot.ci.type = "bca.simple")
pe_med[pe_med$lhs %in% c("indirect", "direct", "total"),
       c("lhs", "est", "se", "z", "pvalue", "ci.lower", "ci.upper")]
##        lhs    est    se      z pvalue ci.lower ci.upper
## 7 indirect  0.042 0.018  2.294  0.022    0.012    0.085
## 8   direct -0.091 0.069 -1.324  0.186   -0.226    0.043
## 9    total -0.049 0.073 -0.668  0.504   -0.195    0.096

The columns:

  • est — the estimate of the defined parameter
  • se — the bootstrap standard error
  • z, pvalue — the Wald statistic and p-value (based on the bootstrap SE)
  • ci.lower, ci.upper — the BCa 95% confidence interval

The key row is indirect. If the BCa 95% CI for the indirect effect does not include zero, the indirect effect is significant at the .05 level — we have evidence of mediation.

Interpreting the results

Read the three defined parameters:

  • indirect (\(ab\)): the mediated effect of con on creat through pa. If the CI excludes zero, mediation is present.
  • direct (\(c'\)): the direct effect of con on creat, controlling for pa. If this is not significant, mediation is full; if it is significant, mediation is partial.
  • total (\(c' + ab\)): the total effect of con on creat. This may or may not be significant — a non-significant total effect does not rule out mediation (see Tutorial 11).
NoteThe total effect can be non-significant even when mediation is present

If the indirect effect is positive and the direct effect is negative (or vice versa), they can cancel each other out, producing a non-significant total effect. This is called inconsistent mediation or a suppressor effect. Always examine the indirect effect directly, not just the total effect.

Worked example 2: Latent SEM mediation

Now we extend the mediation model to use latent variables for all three constructs. This is the full SEM version, combining the measurement model from Tutorial 10 with the mediation structure from Tutorial 11.

The model

# Build parcels for job control (6 items -> 3 parcels)
ps$p1 <- (ps$con9 + ps$con10) / 2
ps$p2 <- (ps$con1 + ps$con8)  / 2
ps$p3 <- (ps$con3 + ps$con6)  / 2

model_med_latent <- '
  # Measurement model
  con   =~ p1 + p2 + p3
  pa    =~ pa1 + pa2 + pa3
  creat =~ creat1 + creat2 + creat3

  # Structural model with labelled paths
  creat ~ b * pa + c * con
  pa    ~ a * con

  # Defined parameters
  indirect := a * b
  direct   := c
  total    := c + (a * b)
'

Fitting with bootstrap

set.seed(1234)
fit_med_latent <- sem(model_med_latent, data = ps,
                      estimator = "ML", se = "boot", bootstrap = 1000)

Fit indices

The latent SEM is overidentified (unlike the path mediation, which was just-identified), so we can examine fit:

fitMeasures(fit_med_latent, c("chisq", "df", "pvalue",
                              "cfi", "tli",
                              "rmsea", "rmsea.ci.lower", "rmsea.ci.upper",
                              "srmr"))
##          chisq             df         pvalue            cfi            tli 
##         25.528         24.000          0.378          0.999          0.998 
##          rmsea rmsea.ci.lower rmsea.ci.upper           srmr

If the fit indices meet the Hu and Bentler cutoffs, we have evidence that the model — both its measurement and structural parts — is consistent with the data.

Writing up the results

An APA-style write-up of the latent SEM mediation:

Structural equation modelling was used to test whether positive affect (PA) mediated the relationship between job control and creativity. Job control was modelled as a latent factor measured by three parcels (p1, p2, p3); PA and creativity were modelled as latent factors measured by their three respective items. The structural paths were estimated with maximum likelihood and 1,000 bootstrap replications; bias-corrected and accelerated (BCa) confidence intervals are reported for the indirect effect.

The model fit the data well, \(\chi^2(24) = ...\), CFI = …, TLI = …, RMSEA = …, 90% CI […, …], SRMR = …. The indirect effect of job control on creativity through PA was \(ab = ...\), 95% BCa CI […, …], indicating significant mediation. The direct effect of job control on creativity was \(c' = ...\), \(p = ...\), suggesting [full/partial] mediation. The total effect was \(c = ...\), \(p = ...\). Positive affect thus [fully/partially] mediated the relationship between job control and creativity.

(Replace the ... with the actual values from the output above.)

NoteReporting bootstrap CIs

When you report a bootstrap CI, state:

  • The number of bootstrap replications (e.g. 1,000)
  • The type of CI (BCa is preferred)
  • The estimate and its CI
  • Whether the CI includes zero

Do not report a p-value for the indirect effect as your primary test — the bootstrap CI is the test. The p-value in the parameterEstimates() output is based on the bootstrap SE and is less accurate than the CI itself.

Common errors

Warninglavaan WARNING: bootstrap only supported with estimator = ML

You requested se = "boot" but left estimator = "MLR" (or did not specify). Change to estimator = "ML". The point estimates are identical; only the standard errors change.

set.seed() not called

Without set.seed(), the bootstrap results will differ every time you run the model. Always set a seed before fitting a bootstrap model so your results are reproducible. We use set.seed(1234) throughout this series.

Very wide bootstrap CIs

If the bootstrap CI for the indirect effect is very wide, it may indicate a small sample, a weak indirect effect, or high variability in the bootstrap samples. Increasing the number of bootstrap replications (e.g. to 5,000) will narrow the CI slightly but will not fix a fundamentally weak effect.

Negative variance (Heywood case) in the latent SEM

If the latent SEM produces a Heywood case — a negative variance for one of the latent factors or indicators — check the measurement model’s fit on its own (fit a CFA first). A poorly-fitting measurement model can distort the structural estimates.

Summary & next tutorial

You can now fit mediation models in lavaan — both path mediation with manifest variables and full latent-variable SEM mediation — using labelled paths and the := operator to define the indirect effect. You can extract BCa bootstrap confidence intervals and report the results in APA format. This completes the core lavaan toolkit: you can specify, estimate, and report CFA, path analysis, SEM, and mediation models. In the next tutorial we turn to the question of reporting — how to export lavaan results to publication-quality tables and figures, with a focus on the texreg package.
