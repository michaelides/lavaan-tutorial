Reporting lavaan results

APA write-ups, exporting tables with texreg, exporting path diagrams

Author
Affiliation

George Michaelides

School of Psychology, University of East Anglia

Published

July 7, 2026

Learning objectives

By the end of this tutorial you will be able to:

  • write an APA-style results paragraph for a CFA, path analysis, SEM, or mediation model
  • export lavaan parameter estimates and fit indices to publication-quality tables with texreg
  • export path diagrams with tidySEM as PNG or PDF files
  • compare nested models with lavTestLRT() and report the comparison
  • assemble a complete results section for a dissertation

Setup

Setup code (you have seen this before) 
library(haven)
library(lavaan)
library(psych)
library(texreg)
library(tidySEM)

The canonical workflow

By now you have fitted CFA, path, SEM, and mediation models across Tutorials 06–12. This tutorial pulls together the reporting side: how to go from a fitted lavaan object to a polished results section for a dissertation or paper.

The canonical workflow is:

  1. Fit the model and check convergence
  2. Extract fit indices with fitMeasures()
  3. Extract parameter estimates with summary() or parameterEstimates()
  4. Export a publication-quality table with texreg
  5. Export a path diagram with tidySEM
  6. Write the APA results paragraph

We will demonstrate each step using the two-factor CFA from Tutorial 07 and the latent SEM mediation from Tutorial 12.

Refitting the models

First, we refit the two models we will use as running examples:

# Two-factor CFA (Tutorial 07)
model_cfa <- '
  PA  =~ pa1 + pa2 + pa3
  neg =~ na1 + na2 + na3
  PA  ~~ neg
'
fit_cfa <- cfa(model_cfa, data = ps, estimator = "MLR")

# Latent SEM mediation (Tutorial 12)
model_sem <- '
  con   =~ p1 + p2 + p3
  pa    =~ pa1 + pa2 + pa3
  creat =~ creat1 + creat2 + creat3

  creat ~ b * pa + c * con
  pa    ~ a * con

  indirect := a * b
  direct   := c
  total    := c + (a * b)
'
set.seed(1234)
fit_sem <- sem(model_sem, data = ps,
               estimator = "ML", se = "boot", bootstrap = 1000)

The model fit the data well, \(\chi^2(df) = value\), \(p = value\), CFI = value, TLI = value, RMSEA = value, 90% CI [lower, upper], SRMR = value. All factor loadings were significant (\(p < .001\)) and substantial (standardised \(\lambda\) range: [low] to [high]). Exporting tables with texreg

The texreg package exports lavaan results to clean, publication-quality tables in three formats: screen (for checking), HTML (for Word documents), and LaTeX (for PDF documents).

Screen output

For a quick check, use screenreg():

                  summaries = c("N", "AIC", "BIC"),
                  labels = list(),

This prints a compact table to the R console with each parameter as a row, its estimate, standard error, and significance stars. The summaries argument controls which fit statistics appear at the bottom.

HTML output (for Word documents)

To insert a lavaan table into a Word document, export it as HTML and copy-paste, or insert the HTML file into Word:

texreg::htmlreg(fit_cfa, file = "tables/table_cfa.html",
                summaries = c("N", "AIC", "BIC"),
                caption = "Confirmatory Factor Analysis of Positive and Negative Affect",
                caption.above = TRUE)

Open the HTML file in a web browser, select all, copy, and paste into Word — the table formatting is preserved.

LaTeX output (for PDF documents)

For documents typeset in LaTeX (e.g. via Quarto with a PDF output):

texreg::texreg(fit_cfa, file = "tables/table_cfa.tex",
               summaries = c("N", "AIC", "BIC"),
               caption = "Confirmatory Factor Analysis of Positive and Negative Affect",
               caption.above = TRUE,
               label = "tab:cfa")

Include the table in a LaTeX or Quarto document with \input{tables/table_cfa.tex} or via Quarto’s $include$ mechanism.

Comparing multiple models

texreg can display several models side by side, which is useful for comparing nested models (e.g. a first-order CFA vs. a second-order CFA, or a path model vs. a full SEM):

texreg::screenreg(list(fit_cfa, fit_cfa),
                  custom.model.names = c("First-order", "Second-order"),
                  summaries = c("N", "AIC", "BIC"))
Notetexreg and lavaan

texreg works with lavaan objects out of the box — it knows how to extract parameter estimates, fit indices, and sample size. For mediation models with := defined parameters, the defined parameters (indirect, direct, total) appear in the table alongside the regression coefficients.

For more complex table layouts, consider semTable (from the semTable package) as an alternative — it is specifically designed for SEM and produces slightly different formatting. We recommend texreg as the default because it handles both lm and lavaan objects, making it useful across your entire dissertation.

Exporting path diagrams

With tidySEM

To save a path diagram as a PNG or PDF, use tidySEM’s graph functions with a graphics device:

# Save as PNG
png("figures/sem_mediation.png", width = 1200, height = 800, res = 150)
tidySEM::graph_sem(fit_sem, std = TRUE)
dev.off()

# Save as PDF (vector graphics, publication quality)
pdf("figures/sem_mediation.pdf", width = 8, height = 6)
tidySEM::graph_sem(fit_sem, std = TRUE)
dev.off()

Comparing nested models

lavTestLRT() for MLR models

When comparing nested models fitted with MLR, use lavTestLRT() rather than anova() — the Satorra–Bentler scaled chi-square difference test is the correct test for robust estimators:

# First-order CFA (Tutorial 07)
fit_1st <- cfa(model_cfa, data = ps, estimator = "MLR")

# Second-order CFA (Tutorial 08)
model_2nd <- '
  PA     =~ pa1 + pa2 + pa3
  neg    =~ na1 + na2 + na3
  affect =~ PA + neg
'
fit_2nd <- cfa(model_2nd, data = ps, estimator = "MLR")

lavTestLRT(fit_1st, fit_2nd)
## 
## Chi-Squared Difference Test
## 
##         Df    AIC    BIC Chisq Chisq diff RMSEA Df diff Pr(>Chisq)
## fit_2nd  7 5248.3 5301.4  9.87                                    
## fit_1st  8 5246.3 5295.6  9.87                0       1

The output reports the scaled chi-square difference, its degrees of freedom, and the p-value. A non-significant result means the more restrictive model (here, the second-order) fits as well as the less restrictive model — the restriction is justified.

anova() for ML models

For models fitted with plain ML (e.g. bootstrap models), anova() is appropriate:

anova(fit_1, fit_2)  # both fitted with estimator = "ML"
WarningUse the right test
  • MLR models → lavTestLRT() (scaled difference test)
  • ML models → anova() (standard likelihood-ratio test)
  • Bootstrap models → no standard difference test; compare bootstrap CIs instead

Using the wrong test gives incorrect p-values. When in doubt, lavTestLRT() is the safer choice because it works for both ML and MLR.

Common errors

WarningReporting a just-identified model’s fit

Do not report “excellent fit” for a just-identified model (df = 0). The fit is guaranteed by the model’s structure. Report the parameter estimates and \(R^2\) values, and note that the model is just-identified.

Using anova() with MLR models

anova() does not apply the Satorra–Bentler scaling correction. Use lavTestLRT() for models fitted with MLR. The two functions give different p-values; only lavTestLRT() is correct for robust estimators.

Forgetting to set the bootstrap seed

If you re-run a bootstrap model without set.seed(), the CIs will differ. Always set the seed, and report the seed in your methods section for full reproducibility.

Summary & next tutorial

You can now write APA-style results paragraphs for CFA, path analysis, SEM, and mediation models; export parameter estimates and fit indices to publication-quality tables with texreg; export path diagrams with tidySEM; and compare nested models with the correct statistical test. This completes the core SEM workflow — you have everything you need for a dissertation using lavaan. In the remaining tutorials we cover advanced topics: multilevel models, growth curve models, and moderation, which extend the lavaan toolkit to nested, longitudinal, and interaction-based analyses.
