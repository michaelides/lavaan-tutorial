CFA — practical

Two-factor CFA on NA/PA; fit, parameter estimates, modification indices

Author
Affiliation

George Michaelides

School of Psychology, University of East Anglia

Published

January 1, 2026

Learning objectives

By the end of this tutorial you will be able to:

  • specify and fit a two-factor CFA in lavaan using the PS dataset
  • read every section of the summary(fit, standardized = TRUE, fit.measures = TRUE) output
  • extract the canonical fit indices with fitMeasures() and judge them against the Hu and Bentler cutoffs
  • compute Cronbach’s alpha for a scale with psych::alpha()
  • request and interpret modification indices with modindices()
  • report CFA results in APA format

Setup

Setup code (you have seen this before) 
library(haven)
library(lavaan)
library(psych)
library(tidySEM)

From Tutorial 06 onward, the setup block — loading packages, importing the PS dataset, and converting missing values — is collapsed behind a <details> tag. Click to expand it if you need to copy the code. The block is the same in every tutorial that uses the PS data.

The model

We fit the two-factor CFA on positive and negative affect that we introduced in Tutorial 06:

  • Positive affect (PA) measured by pa1, pa2, pa3
  • Negative affect (neg) measured by na1, na2, na3
  • The two factors are allowed to correlate

The lavaan syntax:

model_pa_na <- '
  PA  =~ pa1 + pa2 + pa3
  neg =~ na1 + na2 + na3
  PA  ~~ neg
'

We use the marker-variable method (lavaan’s default) to set each factor’s scale: the first loading of each factor is fixed to 1. We estimate with MLR — our default throughout the series.

Fitting the model

fit_pa_na <- cfa(model_pa_na, data = ps, estimator = "MLR")

The model converges silently — no warnings, which means lavaan found a stable solution. If it had not converged, we would see a warning and would need to diagnose the problem (see the Common errors section at the end).

Estimator and optimisation

The header tells us the estimator (MLR), the optimisation method (NLMINB), the number of iterations (typically 20–40 for a well-behaved model), and the number of observations used. lavaan uses FIML by default, so all 343 participants are retained even if some have missing values on individual items.

Model fit indices

The Model Fit Information section reports the canonical set:

  • Chi-Square Test of Model Fit: \(\chi^2\) = 7.927, df = 8, p = .441. The \(\chi^2\) is non-significant, meaning we cannot reject the null hypothesis that the model-implied covariance matrix equals the observed one. This is good news for our model. The Scaling correction factor is reported because we are using MLR — the \(\chi^2\) has been corrected for non-normality.

  • RMSEA: 0.000, 90% CI [0.000, 0.068]. The point estimate is at the floor (zero), and the upper bound of the CI is below .08. This indicates close fit.

  • CFI/TLI: CFI = 1.000, TLI = 1.000. Both are at the maximum, well above the .95 threshold.

  • SRMR: 0.014. Well below the .08 threshold.

All five indices agree: the model fits the data very well. We will report this in APA format at the end of the tutorial.

NoteWhy is the \(\chi^2\) starred?

The asterisk next to the \(\chi^2\) value in MLR output indicates that the standard \(\chi^2\) difference test does not apply directly to robust estimators. If you need to compare nested models fitted with MLR, use lavTestLRT() (the scaled likelihood-ratio test) rather than anova(). We cover this in Tutorial 13.

Parameter estimates

The Parameter Estimates section has several subsections. Let us walk through each.

Latent variables (loadings)

The Estimate column gives the unstandardised factor loading (\(\lambda\)). The first loading of each factor (pa1 and na1) is fixed to 1.000 — the marker-variable method. These fixed loadings have no standard error or p-value, which is why their cells are blank.

The std.all column gives the fully standardised loadings — the correlation between each indicator and its factor. All six loadings are high (0.76–0.85), indicating that each item is a strong indicator of its underlying factor. A loading below .40 would be concerning; below .30 suggests the item is not measuring the factor well.

All freely-estimated loadings have p < .001, meaning each is significantly different from zero.

Covariances

Covariances:
                   Estimate    Std.Err  z value  P(>|z|)   std.all
  PA ~~
    neg             -0.419       0.077   -5.428    0.000   -0.471

The covariance between PA and neg is -0.419. The std.all value (-0.471) is the correlation between the two factors — positive and negative affect are moderately negatively related, which makes theoretical sense. The p < .001 tells us this correlation is significantly different from zero.

Intercepts

Intercepts:

   .pa2              3.421       0.075   45.695    0.000    2.544

The intercepts are the estimated means of the observed variables. They are not usually of substantive interest in CFA — they tell you the average response on each item — but they are part of the model. The dot prefix (.pa1) indicates that this is an intercept for an observed variable that is dependent on a latent variable.

Variances

Variances:
                   Estimate    Std.Err  z value  P(>|z|)   std.all
    PA               0.891       0.122    7.312    0.000    1.000
    neg              0.885       0.098    9.011    0.000    1.000
   .pa1              0.498       0.089    5.620    0.000    0.358
   .pa2              0.673       0.076    8.819    0.000    0.416
    ...

The first two rows are the latent variances (\(\psi\)) — the variance of each factor. The std.all is 1.000 because a variable’s standardised variance is always 1.

The remaining rows are the residual variances (\(\theta\)) — the portion of each indicator’s variance not explained by the factor. The std.all for a residual variance is the proportion of the indicator’s variance that is not explained by the factor. For pa1, that is 0.358, meaning \(1 - 0.358 = 0.642\) (or 64.2%) of the variance in pa1 is explained by the PA factor. This is the indicator’s \(R^2\).

Extracting fit indices cleanly

For reporting, we extract just the canonical set with a named fitMeasures() call:

fit_pa_na_indices <- fitMeasures(fit_pa_na,
                                 c("chisq", "df", "pvalue",
                                   "cfi", "tli",
                                   "rmsea", "rmsea.ci.lower", "rmsea.ci.upper",
                                   "srmr"))

Modification indices

Modification indices (MI) estimate how much the model \(\chi^2\) would improve if a currently-fixed parameter were freely estimated. An MI above 3.84 corresponds to \(\alpha = .05\); above 6.63, to \(\alpha = .01\).

modindices(fit_pa_na, minimum.value = 3.84, sort. = TRUE)
##    lhs op rhs    mi    epc sepc.lv sepc.all sepc.nox
## 31 pa3 ~~ na1 7.036 -0.084  -0.084   -0.224   -0.224

The output lists every fixed parameter whose MI exceeds 3.84, sorted from largest to smallest. The columns are:

  • lhs, op, rhs — the parameter that would be freed (e.g. pa2 ~~ na1 means “free the residual covariance between pa2 and na1”)
  • mi — the modification index (estimated drop in \(\chi^2\) if this parameter were freed)
  • epc — the expected parameter change (what the parameter’s value would be if freed)
  • sepc.lv, sepc.all, sepc.nox — standardised versions of the EPC
WarningUse modification indices cautiously

Modification indices are data-driven suggestions. They tell you where the sample covariance matrix departs from the model, but they do not tell you why. Freeing a parameter because it has a high MI can improve fit in your sample while making the model worse in the population — this is overfitting. Always have a theoretical reason for freeing a parameter, and report any model changes you make transparently. We recommend using MIs as diagnostic information, not as an automatic model-building tool.

In our two-factor CFA, the modification indices are all small (the largest is below 5), which is consistent with the model’s already-excellent fit. There is no compelling reason to free any additional parameter.

Drawing the path diagram

tidySEM::graph_sem() produces a path diagram directly from the fitted lavaan object:

tidySEM::graph_sem(fit_pa_na, std = TRUE)

The diagram shows:

  • Two circles (latent variables: PA, neg)
  • Six squares (observed indicators)
  • Arrows from each factor to its indicators, labelled with the standardised loadings
  • A double-headed arrow between the two factors, labelled with their correlation (-0.47)

This is the same diagrammatic convention we used in Tutorial 06’s concept diagram, but now it is generated from the fitted model — the numbers on the arrows are the actual estimated loadings, not placeholders.

NotetidySEM vs hand-authored diagrams

Throughout this series we use two diagramming tools:

  • tidySEM::graph_sem(fit) — generates a diagram from a fitted lavaan object. The numbers are the actual estimates. Use this when you want to show the results of a fitted model.
  • DiagrammeR::grViz() — hand-authored diagrams for illustrating concepts before a model is fitted. Use this when you want to explain a model structure without tying it to specific estimates.

We used DiagrammeR for the concept diagrams in Tutorials 05 and 06; from Tutorial 07 onward we use tidySEM whenever we want to visualise a fitted model.

Writing up the results

An APA-style write-up of this CFA would look something like:

Confirmatory factor analysis was used to test the hypothesised two-factor structure of affect, with positive affect (PA) measured by three items (pa1, pa2, pa3) and negative affect (neg) measured by three items (na1, na2, na3). The model was estimated with robust maximum likelihood (MLR) to correct for non-normality. The model fit the data well, \(\chi^2(8) = 7.93\), \(p = .441\), CFI = 1.00, TLI = 1.00, RMSEA = 0.00, 90% CI [0.00, 0.07], SRMR = 0.01. All factor loadings were significant (\(p < .001\)) and substantial (standardised \(\lambda\) range: .76–.85). The correlation between PA and neg was \(r = -.47\), \(p < .001\). Cronbach’s alpha was .78 for PA and .82 for neg, indicating acceptable internal consistency for both scales.

NoteReporting conventions

In APA format:

  • Report \(\chi^2\) with degrees of freedom and p-value: \(\chi^2(8) = 7.93\), \(p = .441\)
  • Report fit indices to two decimal places: CFI = 1.00, RMSEA = 0.00
  • Report RMSEA with its 90% CI: RMSEA = 0.00, 90% CI [0.00, 0.07]
  • Report standardised loadings with the symbol \(\lambda\) or just “standardised loading”
  • Report correlations with \(r\): \(r = -.47\)
  • Suppress leading zeros for values that cannot exceed 1: \(r = -.47\), not \(r = -0.47\); \(p = .001\), not \(p = 0.001\)

Common errors

Warninglavaan WARNING: model has not converged

The estimation algorithm could not find a stable solution. Possible causes: model misspecification, underidentification, empirical underidentification (two items nearly identical), or a Heywood case. Check summary(fit) for negative variances or loadings above 1, check that every factor has at least three indicators, and check the correlation matrix of your indicators for near-perfect collinearity.

lavaan WARNING: some estimated lv variances are negative

A Heywood case — a negative variance, which is impossible. Usually indicates that the model is misspecified or the sample is too small. Try constraining the problematic variance to a small positive value, or respecify the model.

Very high modification indices on residual covariances

If modindices() suggests large residual covariances among items with similar wording, you may have common method variance. Consider freeing one or two theoretically-justified residual covariances, but report the change transparently.
