R basics

Objects, vectors, data frames, factors, missing values

Author
Affiliation

George Michaelides

School of Psychology, University of East Anglia

Published

January 1, 2026

Learning objectives

By the end of this tutorial you will be able to:

  • assign values to named objects and inspect what is in your R environment
  • create vectors of different types (numeric, character, logical) and index them
  • build data frames from vectors and extract rows, columns, and cells
  • compute descriptive statistics (mean, median, standard deviation) and handle missing values with na.rm = TRUE
  • create factors for categorical variables and understand why they matter for Likert-scale items

Setup

This tutorial uses only base R — no additional packages are needed. We set the output comment prefix to ## so that the rendered output matches what you will see in your RStudio console.

Assignment and objects

In R, you store values in named objects using the assignment operator <-. Think of an object as a labelled box that holds a value you can refer to later.

my_age <- 34
my_age
## [1] 34

The arrow points from the value on the right to the name on the left, so “34 goes into my_age”. You can also use = for assignment, but <- is the convention we follow throughout these tutorials because it makes the direction of assignment explicit.

Object names should be descriptive and use snake_case — lowercase letters with underscores separating words. Avoid dots in names; they are an older R convention and can be confused with method calls in other languages.

mean_score <- 3.45
participant_id <- "P001"
both_assigned <- TRUE

To see everything in your current environment, use ls(). To remove an object, use rm().

ls()
## [1] "both_assigned"   "mean_score"      "my_age"          "pandoc_dir"     
## [5] "participant_id"  "quarto_bin_path"
rm(both_assigned)
Note

R is case-sensitive. My_Age and my_age are two different objects. If you get an object 'x' not found error, check your capitalisation first.

Vectors

A vector is the fundamental data structure in R — a sequence of values that all share the same type. You create vectors with c(), which stands for “combine” or “concatenate”.

scores <- c(3, 4, 2, 5, 4, 3, 5, 2, 4, 3)
scores
##  [1] 3 4 2 5 4 3 5 2 4 3

Every value in a vector must be the same type. R has four main types you will meet:

age_group      <- c(1, 2, 3, 4, 5)        # numeric (double)
participant    <- c("P001", "P002", "P003") # character
consent_given  <- c(TRUE, TRUE, FALSE, TRUE) # logical
likert_integer <- c(1L, 2L, 3L, 4L, 5L)   # integer (the L suffix forces integer)

You can inspect the type and length of any object with class(), typeof(), and length():

class(scores)
## [1] "numeric"
typeof(scores)
## [1] "double"
length(scores)
## [1] 10

For a compact overview of an object’s structure, use str():

str(scores)
##  num [1:10] 3 4 2 5 4 3 5 2 4 3
TipTry it

Create a character vector called item_names containing the names of three survey items (e.g. "con1", "con3", "con6"). Then check its class and length.

CautionSolution 
item_names <- c("con1", "con3", "con6")
class(item_names)
## [1] "character"
length(item_names)
## [1] 3

Coercion

If you mix types in a vector, R silently coerces everything to the most general type:

mixed <- c(1, "two", 3)
mixed
## [1] "1"   "two" "3"
class(mixed)
## [1] "character"

The number 1 and 3 have been converted to text. This is rarely what you want, so keep your vectors homogeneous.

Indexing vectors

You pull out individual elements of a vector with square brackets. Position numbers start at 1 in R (not 0, as in Python or C).

scores[1]         # first element
## [1] 3
scores[c(1, 3)]   # first and third
## [1] 3 2
scores[2:5]       # second through fifth
## [1] 4 2 5 4

You can also use a logical vector to select elements that meet a condition:

scores[scores > 3]
## [1] 4 5 4 5 4

This is called logical indexing and you will use it constantly. The comparison scores > 3 produces a logical vector of the same length as scores:

scores > 3
##  [1] FALSE  TRUE FALSE  TRUE  TRUE FALSE  TRUE FALSE  TRUE FALSE

R returns TRUE for each element that satisfies the condition, and the square brackets keep only those positions.

TipTry it

Use logical indexing to extract only the scores equal to 5 from scores.

CautionSolution 
scores[scores == 5]
## [1] 5 5

Data frames

A data frame is R’s version of a spreadsheet — a collection of columns (vectors) where each column can have a different type but every column has the same length. This is the structure you will work with most often when analysing survey data.

You can build a data frame from vectors with data.frame():

df <- data.frame(
  participant = c("P001", "P002", "P003", "P004"),
  age_group   = c(2, 3, 4, 3),
  pa_score    = c(4.2, 3.8, 2.1, 3.5)
)
df
##   participant age_group pa_score
## 1        P001         2      4.2
## 2        P002         3      3.8
## 3        P003         4      2.1
## 4        P004         3      3.5

Inspect its dimensions and column names:

nrow(df)
## [1] 4
ncol(df)
## [1] 3
dim(df)
## [1] 4 3
names(df)
## [1] "participant" "age_group"   "pa_score"

For a quick look at the first or last few rows, use head() and tail():

head(df, 2)
##   participant age_group pa_score
## 1        P001         2      4.2
## 2        P002         3      3.8
tail(df, 1)
##   participant age_group pa_score
## 4        P004         3      3.5

And for summary statistics of every column at once, summary():

summary(df)
##     participant   age_group       pa_score   
##  Length   :4    Min.   :2.00   Min.   :2.10  
##  N.unique :4    1st Qu.:2.75   1st Qu.:3.15  
##  N.blank  :0    Median :3.00   Median :3.65  
##  Min.nchar:4    Mean   :3.00   Mean   :3.40  
##  Max.nchar:4    3rd Qu.:3.25   3rd Qu.:3.90  
##                 Max.   :4.00   Max.   :4.20
Note

summary() is your fastest first look at any dataset. It shows the minimum, first quartile, median, mean, third quartile, and maximum for numeric columns, and counts for factors.

Indexing data frames

Data frames use row, column indexing with square brackets:

df[, "pa_score"]          # all rows, one column (returns a vector)
## [1] 4.2 3.8 2.1 3.5
df[, c("participant", "pa_score")]  # two columns (returns a data frame)
##   participant pa_score
## 1        P001      4.2
## 2        P002      3.8
## 3        P003      2.1
## 4        P004      3.5
df[1, ]                   # first row, all columns
##   participant age_group pa_score
## 1        P001         2      4.2
df[1:3, c("participant", "age_group")]  # rows 1-3, two columns
##   participant age_group
## 1        P001         2
## 2        P002         3
## 3        P003         4

The $ operator is a convenient shortcut for extracting a single column by name:

df$pa_score
## [1] 4.2 3.8 2.1 3.5
df$age_group
## [1] 2 3 4 3
TipTry it

Extract the pa_score values for participants whose age_group is 3 or higher, using logical indexing on df$age_group.

CautionSolution 
df$pa_score[df$age_group >= 3]
## [1] 3.8 2.1 3.5

Functions for describing data

R provides built-in functions for the descriptive statistics you will need throughout these tutorials:

mean_scores <- c(3.2, 4.1, 2.8, 3.9, 4.5, 3.1, 2.9, 3.7)

mean(mean_scores)
## [1] 3.525
median(mean_scores)
## [1] 3.45
sd(mean_scores)
## [1] 0.6158618
var(mean_scores)
## [1] 0.3792857
quantile(mean_scores, probs = c(0.25, 0.50, 0.75))
##  25%  50%  75% 
## 3.05 3.45 3.95
range(mean_scores)
## [1] 2.8 4.5

Dealing with NA

Real survey data almost always has missing values. In R, missing values are represented by NA. The problem is that many functions return NA if any NA is present:

data_with_na <- c(3, 4, NA, 2, 5)
mean(data_with_na)
## [1] NA

Pass na.rm = TRUE to tell R to ignore missing values before computing:

mean(data_with_na, na.rm = TRUE)
## [1] 3.5
sd(data_with_na, na.rm = TRUE)
## [1] 1.290994
Warning

Forgetting na.rm = TRUE is the single most common error in early R use. If you see NA as your result and did not expect it, add na.rm = TRUE and try again.

Factors

A factor is R’s structure for categorical variables — a vector of integer codes, each mapped to a text label. Factors matter because many statistical functions (ANOVA, regression with grouping variables) treat factors differently from numeric vectors.

gender_raw <- c(0, 1, 1, 0, 1)
gender <- factor(gender_raw, levels = c(0, 1), labels = c("Male", "Female"))
gender
## [1] Male   Female Female Male   Female
## Levels: Male Female
class(gender)
## [1] "factor"
levels(gender)
## [1] "Male"   "Female"

For Likert-scale items (e.g. 1 = Never, 5 = Very often), you will usually keep them as numeric values because you want means and standard deviations. But for grouping variables like gender, age band, or experimental condition, convert them to factors so R knows to treat them as categories.

table(gender)
## gender
##   Male Female 
##      2      3
Note

You will see in Tutorial 03 that when you import an SPSS .sav file with haven::read_sav(), categorical variables come in as haven_labelled objects. You convert them to factors with haven::as_factor().

Missing values

R distinguishes several kinds of “nothing”:

  • NA — “Not Available”. A missing value. The most common one in survey data.
  • NaN — “Not a Number”. The result of an undefined mathematical operation, e.g. 0 / 0.
  • NULL — “empty”. The object itself does not exist (different from NA, which means the value is missing).

You test for missing values with is.na(), which returns a logical vector:

survey_item <- c(4, 3, NA, 5, 2, NA, 4)
is.na(survey_item)
## [1] FALSE FALSE  TRUE FALSE FALSE  TRUE FALSE
sum(is.na(survey_item))   # count of missing values
## [1] 2

To keep only the complete cases (rows with no NA anywhere), use complete.cases():

complete <- complete.cases(survey_item)
survey_item[complete]
## [1] 4 3 5 2 4
TipTry it

Create a vector items <- c(3, NA, 4, 5, NA, 2, 4). Calculate (a) the number of missing values, (b) the mean of the non-missing values, and (c) a new vector with the NAs removed.

CautionSolution 
items <- c(3, NA, 4, 5, NA, 2, 4)
sum(is.na(items))                     # (a) count missing
## [1] 2
mean(items, na.rm = TRUE)             # (b) mean of non-missing
## [1] 3.6
items[complete.cases(items)]          # (c) remove NAs
## [1] 3 4 5 2 4
NoteVectors vs lists

A vector holds values of the same type. A list can hold values of different types — including other vectors, data frames, or even other lists. You index lists with double square brackets [[ ]] or the $ operator. You will encounter lists when fitting lavaan models (a fitted model object is a list containing coefficients, residuals, and fit indices). For now, know the distinction: c() creates a vector; list() creates a list.

Common errors

WarningObject not found

Error: object 'score' not found — you have either not run the line that creates the object, or you have misspelled the name (remember R is case-sensitive).

Could not find function

Error in foo(): could not find function "foo" — the function does not exist, or the package that provides it has not been loaded with library(). In Tutorial 02 we cover package management in detail.

Unexpected NA in results

Your result is NA when you expected a number. Add na.rm = TRUE to the function call (see above).
